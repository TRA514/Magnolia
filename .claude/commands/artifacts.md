# /project:artifacts

## List All Generated Artifacts

List all artifacts generated by the Shipping Greatness workflow for a product initiative.

## Step 1: Locate the Package Folder

All artifacts live in package folders:

```
datasets/product/packages/{YYYY}/{slug}/
```

- List all existing package folders under `datasets/product/packages/`
- If multiple packages exist, list artifacts for each (or ask PM which one)
- If no packages exist, report that no Shipping Greatness workflows have been started

## Step 2: List Artifacts

For the selected package folder, list every file with:
- File path (relative to repo root)
- Artifact type (Context Brief, Press Release, FAQ, API Design, PRD, etc.)
- Phase it belongs to
- Last modified date
- Brief one-line summary of contents (read the first few lines)

## Output Format

```
## Product Artifacts: {Product Name}

**Package folder**: `datasets/product/packages/{YYYY}/{slug}/`

### Phase 1: Discovery
- `{package}/context-brief.md` — {one-line summary} — Last modified: {date}

### Phase 2: Vision
- `{package}/press-release-external.md` — {one-line summary} — Last modified: {date}
- `{package}/press-release-internal.md` — {one-line summary} — Last modified: {date}
- `{package}/one-pager.md` — {one-line summary} — Last modified: {date}

### Phase 3: Knowledge Base
- `{package}/living-faq.md` — {one-line summary} — Last modified: {date}
- `{package}/ai-agent-scenarios.md` — {one-line summary} — Last modified: {date}

### Phase 4: PRD
- `{package}/PRD_{slug}.md` — {one-line summary} — Last modified: {date}
- `datasets/product/prds/{YYYY}/PRD_{slug}.md` — (canonical PRD copy)
- `{package}/expansion-proposals.md` — {one-line summary} — Last modified: {date}

### Phase 5: Validation
- `{package}/red-team-report.md` — {one-line summary} — Last modified: {date}

### Phase 6: Business Case
- `{package}/business-case-swag.md` — {one-line summary} — Last modified: {date}
```

If an artifact doesn't exist, omit it from the list (use `/project:status` to see what's missing).
